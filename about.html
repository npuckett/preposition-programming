JavaScript Objects vs Individual Variables

One of the powerful features of JavaScript is its ability to create objects that group related information together. This makes code much more organized and easier to understand.

The Object-Based Approach (Recommended)

In our "Between" example, we define circles as objects that contain all their properties:

Clean and Organized:

let blueCircle = {
  x: 100,
  y: 150,
  radius: 25,
  dragging: false
};

let redCircle = {
  x: 300,
  y: 150,
  radius: 25,
  dragging: false
};

let greenCircle = {
  x: 200,
  y: 150,
  radius: 20,
  dragging: false
};

The Individual Variable Approach (Harder to Manage)

Without objects, you'd need separate variables for every property of every circle:

Cluttered and Confusing:

let blueCircleX = 100;
let blueCircleY = 150;
let blueCircleRadius = 25;
let blueCircleDragging = false;

let redCircleX = 300;
let redCircleY = 150;
let redCircleRadius = 25;
let redCircleDragging = false;

let greenCircleX = 200;
let greenCircleY = 150;
let greenCircleRadius = 20;
let greenCircleDragging = false;

Why Objects Are Better

Compare how you'd use these variables in your code:

With Objects (Clear):

ellipse(blueCircle.x, blueCircle.y, blueCircle.radius * 2);
if (blueCircle.dragging) { /* move the circle */ }

With Individual Variables (Messy):

ellipse(blueCircleX, blueCircleY, blueCircleRadius * 2);
if (blueCircleDragging) { /* move the circle */ }

Key Benefits of Objects

  • Organization: Related data stays together
  • Clarity: blueCircle.x is clearer than blueCircleX
  • Flexibility: Easy to add new properties to objects
  • Reusability: You can create functions that work with any circle object
  • Less Typing: Shorter, more readable variable names

Real-World Example

In the "Between" example, when we check if a circle is being clicked, compare these approaches:

With Objects:

if (dist(mouseX, mouseY, blueCircle.x, blueCircle.y) < blueCircle.radius) {
  blueCircle.dragging = true;
}

With Individual Variables:

if (dist(mouseX, mouseY, blueCircleX, blueCircleY) < blueCircleRadius) {
  blueCircleDragging = true;
}

The object-based approach makes it immediately clear that all these values belong to the same blue circle, while the individual variables require you to mentally group them together.

When You'll Use This

As you build more complex programs, you'll often need to manage multiple related pieces of information. Objects help you:

  • Keep track of game characters with position, health, and abilities
  • Manage user interface elements with size, color, and behavior
  • Store data about real-world things like students, products, or locations
  • Organize any collection of related properties and actions
